O Projeto Final — O que é e por que importa

💡
O Projeto Final de TIC é sua oportunidade de integrar TUDO que aprendeu ao longo do ano em algo real e útil.

O que seu projeto deve ter:
• Resolver um problema real ou comunicar uma ideia importante
• Usar pelo menos 3 das ferramentas aprendidas (Docs, Sheets, Slides, IA)
• Ser apresentável para uma audiência (turma, família, empresa)
• Demonstrar seu crescimento durante o ano

Tipos de projeto possíveis:
• Campanha social: conscientização sobre um problema + materiais visuais + apresentação
• Guia/Manual: documento completo ensinando algo que você domina
• Análise de dados: planilha com dados reais + gráficos + relatório + apresentação
• Portfólio digital: seu trabalho do ano organizado profissionalmente
• Produto com IA: usa múltiplas ferramentas de IA para criar algo criativo

Como Definir um Bom Projeto — Framework SMART

📋
Um bom projeto tem objetivos SMART:

S — Specific (Específico):
❌ "Fazer algo sobre IA" → ✅ "Criar um guia visual de 10 páginas sobre como usar ChatGPT para estudar"

M — Measurable (Mensurável):
"Terá 10 páginas, 5 seções, 3 exemplos práticos por seção"

A — Achievable (Alcançável):
Pode ser feito com as ferramentas disponíveis nas semanas que temos

R — Relevant (Relevante):
Resolve um problema real, é útil para alguém

T — Time-bound (Com prazo):
"Entregue na Semana 37" — com marcos intermediários nas semanas 33, 34, 35

Teste: se você não consegue preencher todos os 5, o objetivo precisa ser refinado!
⚡ +10 XP
✏️ Framework para definir objetivos de projeto: Específico, Mensurável, Alcançável, Relevante e com Prazo — sigla:

Trabalho em Equipe — Papéis e Responsabilidades

👥
Para projetos em grupo, definir papéis evita conflitos e garante qualidade:

Papéis comuns em projetos de TIC:
• Gerente de Projeto: coordena o time, acompanha prazos, resolve bloqueios
• Designer/Visual: responsável pelo visual: slides, imagens, layout de documentos
• Redator/Conteúdo: pesquisa e escreve o conteúdo textual
• Analista de Dados: responsável pela planilha, gráficos e análise quantitativa
• Especialista IA: integra ferramentas de IA ao projeto

Regras de ouro para trabalho em grupo:
• Cada pessoa tem entregas claras com prazo
• Uma reunião rápida (15 min) por semana para alinhamento
• Documento compartilhado no Google Drive com tudo
• Conflitos resolvidos na reunião, não por mensagem
⚡ +10 XP
✏️ Papel no projeto responsável por coordenar o time e acompanhar prazos chama-se Gerente de:

Estrutura do Projeto — Entregáveis por Semana

📐
Para garantir que o projeto seja concluído no prazo, divide em entregáveis semanais:

Semana 31 (esta semana):
→ Definir tema, equipe e objetivo SMART
→ Criar documento de briefing
→ Atribuir papéis e responsabilidades

Semana 32:
→ Estrutura detalhada (outline completo)
→ Cronograma com tarefas por pessoa
→ Primeiros rascunhos de conteúdo

Semanas 33-35:
→ Desenvolvimento: criar todos os componentes
→ Integração: juntar tudo de forma coesa
→ Revisão: feedback dos colegas e professor

Semanas 36-37:
→ Refinamento e apresentação final
