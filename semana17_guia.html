📝 Guia de Exercícios — Semana 17
📋 Fluxogramas e Pseudocódigo 2º Bimestre 3 exercícios

Este guia contém as instruções passo a passo, a rubrica de avaliação detalhada e os critérios de qualidade para cada exercício da Semana 17. Siga os passos na ordem indicada e use a rubrica para autoavaliar seu trabalho antes de entregar.

1
Criar Fluxogramas de Algoritmos Cotidianos
📊 Fluxogramas ⏱ 35 min

Crie fluxogramas corretos para 3 algoritmos do cotidiano usando os símbolos padronizados.

📋 Passo a Passo
  1. Abra o Google Docs. Use Inserir → Desenho → Novo para criar cada fluxograma
  2. FLUXOGRAMA 1: Verificar se deve levar guarda-chuva (decisão baseada em previsão do tempo)
  3. FLUXOGRAMA 2: Escolher o que comer no almoço (com decisão de orçamento e ingredientes disponíveis)
  4. FLUXOGRAMA 3: Algoritmo do seu próprio dia (da hora que acorda até sair para a escola)
  5. Use os símbolos corretos: oval=início/fim, retângulo=ação, losango=decisão, paralelogramo=E/S
  6. Cada fluxograma deve ter pelo menos 2 decisões (losangos)
✅ Rubrica de Avaliação (Total: 100 pts)
  • 3 fluxogramas criados (30 pts)
  • Símbolos corretos utilizados (30 pts)
  • 2+ decisões em cada fluxograma (20 pts)
  • Fluxos lógicos e sem ambiguidade (20 pts)
2
Escrever Pseudocódigo Correto
📝 Pseudocódigo ⏱ 25 min

Converta 4 situações do cotidiano em pseudocódigo estruturado com INICIO, LEIA, cálculo e ESCREVA.

📋 Passo a Passo
  1. ALGORITMO 1: Calcular IMC (peso/altura²) e classificar
  2. ALGORITMO 2: Sistema de login simples (3 tentativas máximas)
  3. ALGORITMO 3: Calculadora de troco (verifica se pagou o suficiente)
  4. ALGORITMO 4: Classificar nota escolar (A/B/C/D/F ou Ótimo/Bom/Regular/Insuficiente)
  5. Para cada algoritmo: use INICIO/FIM, LEIA para entradas, ESCREVA para saídas
  6. Teste manualmente cada algoritmo com 3 conjuntos de valores diferentes
✅ Rubrica de Avaliação (Total: 100 pts)
  • 4 pseudocódigos com estrutura correta (40 pts)
  • LEIA e ESCREVA usados adequadamente (20 pts)
  • Lógica correta em todos os algoritmos (25 pts)
  • Testes manuais documentados (15 pts)
3
Algoritmo do GPS — Análise e Simulação
🗺️ Algoritmos Reais ⏱ 25 min

Analise o algoritmo de Dijkstra para encontrar caminhos mínimos e simule manualmente em um grafo simples.

📋 Passo a Passo
  1. Crie um mapa fictício de 6 cidades com distâncias entre elas (no Google Docs com formas)
  2. Aplique o algoritmo de Dijkstra manualmente: partindo da Cidade A, encontre o menor caminho para todas as outras
  3. Documente cada passo: quais cidades foram visitadas, quais distâncias foram calculadas
  4. Pesquise: como o GPS real usa variações deste algoritmo? (considera tráfego, velocidade, etc.)
  5. Escreva o pseudocódigo simplificado do algoritmo de Dijkstra
✅ Rubrica de Avaliação (Total: 100 pts)
  • Mapa com 6 cidades criado (20 pts)
  • Dijkstra executado corretamente (35 pts)
  • Cada passo documentado (20 pts)
  • Pesquisa sobre GPS real (15 pts)
  • Pseudocódigo do Dijkstra (10 pts)
