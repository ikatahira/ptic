Integração — Juntar as Partes em um Todo Coeso

🔗
Cada membro criou suas partes. Agora é hora de integrar — garantir que tudo flua como um projeto único.

Problemas comuns na integração:
• Partes com estilos visuais diferentes
• Terminologia inconsistente (um usou "aluno", outro "estudante")
• Fluxo quebrado — leitor vai de A para C sem entender B
• Tamanho desproporcionado das seções

Como integrar bem:
1. Uma pessoa lê o projeto do início ao fim como se fosse leitor pela primeira vez
2. Identifica onde o fluxo quebra ou confunde
3. Ajusta transições entre seções
4. Padroniza terminologia e estilo
5. Verifica tamanho das seções — tudo proporcional?

Google Docs tem o recurso "Sugestões": ative o modo Sugestão para propor edições sem alterar o original

Refinamento — A Diferença entre Bom e Excelente

✨
A diferença entre um projeto bom e um excelente está nos detalhes de refinamento:

Refinamento de Texto:
• Use o verificador ortográfico do Docs/Word (pressione F7)
• Peça ao ChatGPT: "Melhore este parágrafo para ser mais claro e profissional"
• Remova redundâncias — cada palavra deve ter propósito
• Verifique: o leitor entende sem saber o contexto?

Refinamento Visual:
• Alinhamentos: use Alinhar ao centro/guias no Canva
• Espaçamento consistente entre elementos
• Imagens de qualidade (mínimo 150 DPI para impressão)
• Contraste adequado (texto escuro em fundo claro)

Refinamento de Dados:
• Gráficos têm título, eixos com unidades, legenda?
• Os insights escritos fazem referência aos dados?
• Os dados estão atualizados e verificados?
⚡ +10 XP
✏️ Para verificar ortografia rapidamente no Word e Google Docs, pressione a tecla:
F7

Preparar a Apresentação Final — Contar uma História

🎤
A apresentação final não é um resumo do projeto — é uma história que convence, emociona e informa.

Estrutura narrativa para apresentações:
1. Gancho (1 min): começa com uma pergunta, dado chocante ou história
"Você sabia que 80% dos estudantes não sabem usar IA produtivamente?"

2. Problema (2 min): por que isso importa? Para quem é esse problema?

3. Solução (5 min): o que vocês criaram e como resolve o problema

4. Evidências (3 min): dados, exemplos, demonstração do projeto

5. Conclusão e próximos passos (1 min): o que vem depois?

Dica de ouro: ensaie em voz alta pelo menos 2 vezes. O que parece bom escrito pode soar estranho quando falado.
⚡ +10 XP
✏️ Primeiro elemento de uma boa apresentação que captura imediatamente a atenção da audiência chama-se:
gancho

Apresentar Dados com Impacto — Data Storytelling

📊
Dados sozinhos não convencem ninguém. Data Storytelling combina dados com narrativa:

Técnica dos 3 passos:
1. Contexto: "Pesquisamos 30 alunos do ETEC sobre hábitos de estudo"
2. Dado: "72% disseram que estudam menos de 1 hora por dia"
3. Insight/Implicação: "Isso explica por que técnicas de estudo eficientes com IA são tão necessárias"

Design do gráfico para impacto:
• Título que já diz a conclusão: "72% dos Alunos Estudam Menos de 1h/Dia"
• Destaque o dado mais importante (cor diferente, maior)
• Remova elementos desnecessários (gridlines excessivas, bordas)
• Anotações direto no gráfico (não só na legenda)

Ferramenta: Google Slides tem gráficos integrados com o Sheets — atualiza automaticamente!
